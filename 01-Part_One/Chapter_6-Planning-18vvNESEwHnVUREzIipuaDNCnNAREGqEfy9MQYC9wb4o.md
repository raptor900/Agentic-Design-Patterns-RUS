# Глава 6: Планирование

Интеллектуальное поведение часто включает нечто большее, чем просто реакцию на непосредственный ввод. Оно требует предусмотрительности, разбиения сложных задач на более мелкие управляемые шаги и выработки стратегии достижения желаемого результата. Именно здесь в игру вступает паттерн «Планирование». В основе своей планирование — это способность агента или системы агентов формулировать последовательность действий для перехода из начального состояния в целевое.

## Обзор паттерна «Планирование»

В контексте ИИ полезно рассматривать агента-планировщика как специалиста, которому вы поручаете сложную цель. Когда вы просите его «организовать корпоративный выезд», вы определяете *что* — цель и ограничения — но не *как*. Основная задача агента — автономно проложить курс к этой цели. Он должен понять начальное состояние (бюджет, количество участников, желаемые даты) и целевое состояние (успешно забронированный выезд), а затем найти оптимальную последовательность действий для их соединения. План заранее неизвестен — он создаётся в ответ на запрос.

Характерная черта этого процесса — адаптивность. Начальный план — лишь отправная точка, а не жёсткий сценарий. Реальная сила агента — в способности включать новую информацию и обходить препятствия. Если предпочтительная площадка становится недоступной или выбранный кейтеринг полностью забронирован, способный агент не просто терпит неудачу — он адаптируется. Он фиксирует новое ограничение, переоценивает варианты и формулирует новый план.

Однако важно признать компромисс между гибкостью и предсказуемостью. Динамическое планирование — специфический инструмент, а не универсальное решение. Когда решение задачи уже хорошо понятно и повторяемо, ограничение агента предопределённым фиксированным рабочим процессом эффективнее. Этот подход снижает автономию агента, уменьшая неопределённость и гарантируя надёжный результат. Решение между использованием агента-планировщика и простого агента-исполнителя сводится к одному вопросу: нужно ли «как» открывать, или оно уже известно?

## Практические применения и сценарии использования

Планирование — базовый вычислительный процесс в автономных системах, позволяющий агенту синтезировать последовательность действий для достижения цели в динамичных или сложных средах.

В **автоматизации процедурных задач** планирование оркестрирует сложные рабочие процессы. Например, процесс адаптации нового сотрудника разбивается на подзадачи: создание учётных записей, назначение обучения, координация с департаментами.

В **робототехнике и автономной навигации** планирование фундаментально для обхода пространства состояний — система генерирует путь или последовательность действий, оптимизируя метрики (время, энергопотребление) при соблюдении ограничений среды.

Для **структурированного синтеза информации** агент формулирует план с фазами сбора данных, обобщения, структурирования и итеративной доработки. В **клиентской поддержке** агент создаёт систематический план диагностики, решения и эскалации.

По сути, планирование позволяет агенту перейти от простых реактивных действий к целенаправленному поведению.

## Практический пример кода (Crew AI)

```python
import os
from dotenv import load_dotenv
from crewai import Agent, Task, Crew, Process
from langchain_openai import ChatOpenAI

load_dotenv()

llm = ChatOpenAI(model="gpt-4-turbo")

planner_writer_agent = Agent(
    role='Article Planner and Writer',
    goal='Plan and then write a concise, engaging summary on a specified topic.',
    backstory=(
        'You are an expert technical writer and content strategist. '
        'Your strength lies in creating a clear, actionable plan before writing, '
        'ensuring the final summary is both informative and easy to digest.'
    ),
    verbose=True,
    allow_delegation=False,
    llm=llm,
)

topic = "The importance of Reinforcement Learning in AI"

high_level_task = Task(
    description=(
        f"1. Create a bullet-point plan for a summary on the topic: '{topic}'.\n"
        f"2. Write the summary based on your plan, keeping it around 200 words."
    ),
    expected_output=(
        "A final report containing two distinct sections:\n\n"
        "### Plan\n"
        "- A bulleted list outlining the main points of the summary.\n\n"
        "### Summary\n"
        "- A concise and well-structured summary of the topic."
    ),
    agent=planner_writer_agent,
)

crew = Crew(
    agents=[planner_writer_agent],
    tasks=[high_level_task],
    process=Process.sequential,
)

print("## Running the planning and writing task ##")
result = crew.kickoff()

print("\n\n---\n## Task Result ##\n---")
print(result)
```

Этот код использует библиотеку CrewAI для создания агента, который сначала планирует, а затем пишет обзор заданной темы. ChatOpenAI инициализируется явно. Агент `planner_writer_agent` получает роль и цель: планировать и писать краткие обзоры. Задача требует сначала создать план, затем написать обзор на тему «Важность обучения с подкреплением в ИИ». Crew обрабатывает задачи последовательно.

## Google DeepResearch

Google Gemini DeepResearch (см. Рис. 1) — агентная система для автономного поиска и синтеза информации. Она функционирует через многошаговый агентный пайплайн, динамически и итеративно запрашивающий Google Search для систематического исследования сложных тем. Система обрабатывает большой корпус веб-источников, оценивает собранные данные на релевантность и пробелы в знаниях, проводит последующие поиски. Финальный результат — структурированный многостраничный обзор с цитированием источников.

Процесс начинается с разложения пользовательского запроса в многоуровневый исследовательский план (см. Рис. 1), который затем предоставляется пользователю для рецензирования и модификации. После утверждения агентный пайплайн запускает итеративный цикл поиска и анализа. Система динамически формулирует и уточняет запросы, выявляя пробелы в знаниях и проверяя данные.

## OpenAI Deep Research API

OpenAI Deep Research API — специализированный инструмент для автоматизации сложных исследовательских задач. Используя продвинутую агентную модель, он автономно рассуждает, планирует и синтезирует информацию из реальных источников. Принимает высокоуровневый запрос, декомпозирует его на подзадачи, выполняет веб-поиск и предоставляет структурированный отчёт с цитированием.

Ключевые преимущества:

* **Структурированный вывод с цитированием:** Хорошо организованные отчёты со встроенными цитатами и метаданными источников.
* **Прозрачность:** API раскрывает все промежуточные шаги: рассуждения модели, поисковые запросы, выполненный код.
* **Расширяемость:** Поддержка Model Context Protocol (MCP) для подключения к проприетарным базам знаний.

```python
from openai import OpenAI

client = OpenAI(api_key="YOUR_OPENAI_API_KEY")

system_message = """
You are a professional researcher preparing a structured, data-driven report.
Focus on data-rich insights, use reliable sources, and include inline citations.
"""

user_query = "Research the economic impact of semaglutide on global healthcare systems."

response = client.responses.create(
    model="o3-deep-research-2025-06-26",
    input=[
        {
            "role": "developer",
            "content": [{"type": "input_text", "text": system_message}],
        },
        {
            "role": "user",
            "content": [{"type": "input_text", "text": user_query}],
        },
    ],
    reasoning={"summary": "auto"},
    tools=[{"type": "web_search_preview"}],
)

final_report = response.output[-1].content[0].text
print(final_report)

print("--- CITATIONS ---")
annotations = response.output[-1].content[0].annotations

if not annotations:
    print("No annotations found in the report.")
else:
    for i, citation in enumerate(annotations):
        cited_text = final_report[citation.start_index : citation.end_index]
        print(f"Citation {i + 1}:")
        print(f"  Cited Text: {cited_text}")
        print(f"  Title: {citation.title}")
        print(f"  URL: {citation.url}")

print("\n--- INTERMEDIATE STEPS ---")

try:
    reasoning_step = next(item for item in response.output if item.type == "reasoning")
    print("\n[Reasoning Step]")
    for summary_part in reasoning_step.summary:
        print(f"  - {summary_part.text}")
except StopIteration:
    print("\nNo reasoning steps found.")

try:
    search_step = next(item for item in response.output if item.type == "web_search_call")
    print("\n[Web Search Call]")
    print(f"  Query Executed: \'{search_step.action[\'query\']}\'")
except StopIteration:
    print("\nNo web search steps found.")
```

Код инициализирует клиент OpenAI и выполняет глубокое исследование. Определяется системное сообщение (роль исследователя) и пользовательский запрос. API-вызов создаётся с моделью `o3-deep-research-2025-06-26`, инструментом `web_search_preview` и автоматическим обобщением рассуждений. После получения ответа извлекается финальный отчёт, цитаты с метаданными, а также промежуточные шаги: рассуждения, поисковые запросы, выполненный код.

## Краткий обзор

**Что:** Сложные задачи нельзя решить одним действием. Без структурированного подхода агентная система не может разбить высокоуровневые цели на управляемые подзадачи.

**Почему:** Планирование позволяет системе сначала создать связный план, разбивая цель на последовательность действий. LLM особенно подходят для генерации планов на основе обучающих данных.

**Когда использовать:** Когда запрос слишком сложен для одного действия или инструмента. Идеально для многошаговых процессов: генерация отчётов, адаптация сотрудников, конкурентный анализ.

* Планирование позволяет агентам разбивать сложные цели на последовательные действия.
* Необходимо для многошаговых задач, автоматизации рабочих процессов и навигации в сложных средах.
* LLM могут планировать, генерируя пошаговые подходы на основе описаний задач.
* Google Deep Research — агент, анализирующий источники с помощью Google Search, с рефлексией, планированием и выполнением.

## Заключение

Паттерн «Планирование» — фундаментальный компонент, переводящий агентные системы от простых реактивных ответов к стратегическим целенаправленным исполнителям. Он масштабируется от простого последовательного выполнения (CrewAI-агент) до сложных динамических систем (Google DeepResearch). Планирование обеспечивает мост между намерением пользователя и автоматизированным выполнением сложных задач.
