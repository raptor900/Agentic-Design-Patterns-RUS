# Глава 13: Человек в контуре (Human-in-the-Loop)

Паттерн «Человек в контуре» (Human-in-the-Loop, HITL) представляет собой ключевую стратегию в разработке и внедрении агентов. Он целенаправленно переплетает уникальные преимущества человеческого познания — суждение, креативность, нюансированное понимание — с вычислительной мощностью и эффективностью ИИ. Эта стратегическая интеграция часто является необходимостью, особенно по мере внедрения AI-систем в критические процессы принятия решений.

Базовый принцип HITL — обеспечение работы ИИ в этических границах, соблюдение протоколов безопасности и достижение целей с максимальной эффективностью. Это особенно важно в доменах, характеризующихся сложностью, неоднозначностью или значительным риском, где последствия ошибок ИИ могут быть существенными.

Подход HITL вращается вокруг идеи синергии искусственного и человеческого интеллекта. Вместо замены человеческого труда ИИ позиционируется как инструмент, расширяющий человеческие возможности.

На практике HITL реализуется различными способами: валидация и рецензирование, активное управление поведением ИИ в реальном времени, совместное решение проблем через интерактивный диалог.

## Обзор паттерна «Человек в контуре»

HITL интегрирует ИИ с человеческим вводом для расширения возможностей агентов. Оптимальная производительность часто требует сочетания автоматизированной обработки и человеческой интуиции.

HITL включает несколько аспектов:

* **Человеческий надзор:** Мониторинг производительности и вывода агента (через логи, дашборды) для соблюдения правил и предотвращения нежелательных результатов.
* **Вмешательство и коррекция:** При ошибках или неоднозначных сценариях агент запрашивает вмешательство человека; оператор исправляет ошибки, восполняет пробелы или направляет агента.
* **Обратная связь для обучения:** Сбор и использование для уточнения моделей (RLHF — обучение с подкреплением с человеческой обратной связью).
* **Аугментация решений:** Агент предоставляет анализ и рекомендации, человек принимает финальное решение.
* **Человеко-агентное сотрудничество:** Кооперативное взаимодействие, где рутинные задачи — агенту, творческое решение — человеку.
* **Политики эскалации:** Протоколы, определяющие когда и как агент должен передавать задачи операторам.

Реализация HITL позволяет использовать агентов в чувствительных секторах, где полная автономия невозможна. Например, в финансах финальное одобрение крупного кредита требует оценки качественных факторов. В праве — принципы справедливости требуют финального авторитета судьи.

> **Оговорки:** Несмотря на преимущества, HITL имеет существенные ограничения. Главное — недостаточная масштабируемость: человеческий надзор даёт высокую точность, но операторы не могут обрабатывать миллионы задач. Эффективность зависит от квалификации операторов. Реализация также ставит вопросы приватности: чувствительные данные должны анонимизироваться перед доступом операторов.

## Практические применения и сценарии использования

* **Модерация контента:** ИИ фильтрует контент, неоднозначные случаи — эскалация модераторам.
* **Беспилотные автомобили:** Автономное управление с передачей контроля человеку в сложных ситуациях.
* **Детекция финансового мошенничества:** ИИ флагирует подозрительные транзакции, аналитики проводят расследование.
* **Юридический анализ документов:** ИИ сканирует и категоризирует, юристы проверяют точность.
* **Поддержка клиентов (сложные запросы):** Чатбот → оператор при сложных или эмоциональных проблемах.
* **Разметка данных:** Люди размечают данные для обучения моделей.
* **Доработка генеративного ИИ:** Редакторы проверяют и улучшают вывод LLM.
* **Автономные сети:** ИИ анализирует алерты, критические решения эскалируются аналитикам.

«Человек на контуре» (Human-on-the-loop) — вариация, где эксперты определяют политику, а ИИ выполняет немедленные действия. Примеры:

* **Автоматизированный трейдинг:** Эксперт задаёт стратегию (70% акций, 30% облигации), ИИ выполняет сделки в реальном времени.
* **Колл-центр:** Менеджер задаёт правила маршрутизации, ИИ обрабатывает звонки автономно.

## Практический пример кода

Для демонстрации HITL агент ADK идентифицирует сценарии, требующие человеческой проверки, и инициирует эскалацию.

```python
from typing import Optional
from google.adk.agents import Agent
from google.adk.tools.tool_context import ToolContext
from google.adk.callbacks import CallbackContext
from google.adk.models.llm import LlmRequest
from google.genai import types


def troubleshoot_issue(issue: str) -> dict:
    return {"status": "success", "report": f"Troubleshooting steps for {issue}."}

def create_ticket(issue_type: str, details: str) -> dict:
    return {"status": "success", "ticket_id": "TICKET123"}

def escalate_to_human(issue_type: str) -> dict:
    return {"status": "success", "message": f"Escalated {issue_type} to a human specialist."}


technical_support_agent = Agent(
    name="technical_support_specialist",
    model="gemini-2.0-flash-exp",
    instruction="""
    You are a technical support specialist for our electronics company.
    FIRST, check if the user has a support history in state["customer_info"]["support_history"].
    If they do, reference this history in your responses.

    For technical issues:
    1. Use the troubleshoot_issue tool to analyze the problem.
    2. Guide the user through basic troubleshooting steps.
    3. If the issue persists, use create_ticket to log the issue.

    For complex issues beyond basic troubleshooting:
    1. Use escalate_to_human to transfer to a human specialist.

    Maintain a professional but empathetic tone.
    """,
    tools=[troubleshoot_issue, create_ticket, escalate_to_human],
)


def personalization_callback(
    callback_context: CallbackContext, llm_request: LlmRequest
) -> Optional[LlmRequest]:
    """Adds personalization information to the LLM request."""
    customer_info = callback_context.state.get("customer_info")
    if customer_info:
        customer_name = customer_info.get("name", "valued customer")
        customer_tier = customer_info.get("tier", "standard")
        personalization_note = (
            f"\nIMPORTANT PERSONALIZATION:\n"
            f"Customer Name: {customer_name}\n"
            f"Customer Tier: {customer_tier}\n"
        )
        if llm_request.contents:
            system_content = types.Content(
                role="system", parts=[types.Part(text=personalization_note)],
            )
            llm_request.contents.insert(0, system_content)
    return None
```

Код предоставляет шаблон техподдержки на Google ADK с HITL. Агент оснащён инструментами: `troubleshoot_issue`, `create_ticket`, `escalate_to_human`. Инструмент эскалации — ключевой компонент HITL. Персонализация реализуется через callback, динамически извлекающий данные клиента из состояния агента и инжектирующий их в промпт.

## Краткий обзор

**Что:** AI-системы, включая продвинутые LLM, часто struggle с задачами, требующими нюансированных суждений, этического рассуждения или глубокого понимания сложных контекстов. Полная автономия в критических средах несёт значительные риски.

**Почему:** HITL стратегически интегрирует человеческий надзор в AI-рабочие процессы. Симбиотическое партнёрство: ИИ — вычисления, человек — критическая валидация и вмешательство.

**Когда использовать:** При внедрении ИИ в домены с серьёзными последствиями ошибок (медицина, финансы). Для задач с неоднозначностью. Для непрерывного улучшения моделей через человеческие данные.

## Ключевые выводы

* HITL интегрирует человеческий интеллект и суждение в AI-рабочие процессы.
* Критически важен для безопасности, этики и эффективности в сложных сценариях.
* Ключевые аспекты: надзор, вмешательство, обратная связь для обучения, аугментация решений.
* Политики эскалации необходимы для передачи задач человеку.
* Главные ограничения: недостаточная масштабируемость, зависимость от квалифицированных экспертов, вопросы приватности.
