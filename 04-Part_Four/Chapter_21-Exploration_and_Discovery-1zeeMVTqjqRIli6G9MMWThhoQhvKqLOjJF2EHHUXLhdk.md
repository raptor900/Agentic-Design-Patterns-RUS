# Глава 21: Исследование и открытие

Эта глава рассматривает паттерны, позволяющие интеллектуальным агентам активно находить новую информацию, открывать новые возможности и выявлять неизвестные неизвестные (unknown unknowns) в своей операционной среде. Исследование и открытие отличаются от реактивного поведения или оптимизации в пределах предопределенного пространства решений. Вместо этого они фокусируются на проактивном выходе агентов в незнакомые территории, экспериментировании с новыми подходами и генерации нового знания. Этот паттерн критически важен для агентов в открытых, сложных или быстро эволюционирующих доменах, где статических знаний недостаточно.

## Практические применения и сценарии использования

* **Автоматизация научных исследований:** Агент проектирует и проводит эксперименты, анализирует результаты, формулирует гипотезы.
* **Игры и генерация стратегий:** Исследование игровых состояний, открытие emergent-стратегий (AlphaGo).
* **Исследование рынка:** Сканирование данных для выявления трендов и возможностей.
* **Поиск уязвимостей:** Пробинг систем для обнаружения слабых мест.
* **Генерация креативного контента:** Исследование комбинаций стилей, тем, данных.
* **Персонализированное обучение:** AI-репетиторы адаптируют пути обучения.

## Google Co-Scientist

AI Co-Scientist — система Google Research, выступающая вычислительным научным сотрудником. Работает на Gemini, помогает в генерации гипотез, доработке предложений и дизайне экспериментов.

Архитектура — многоагентная система с супервизором, управляющим специализированными агентами (см. Рис. 1):

* **Generation agent:** Генерирует начальные гипотезы через обзор литературы и симулированные научные дебаты.
* **Reflection agent:** Критически оценивает корректность, новизну и качество гипотез.
* **Ranking agent:** Ранжирует гипотезы через турнирную систему на основе Elo.
* **Evolution agent:** Непрерывно совершенствует top-гипотезы.
* **Proximity agent:** Строит граф близости для кластеризации идей.
* **Meta-review agent:** Синтезирует инсайты и обеспечивает обратную связь.

**Валидация:** На бенчмарке GPQA система достигла top-1 точности 78.4%. Scaling test-time compute consistently improves quality. В биомедицине: предложены новые кандидаты лекарств для AML (KIRA6), выявлены эпигенетические мишени для фиброза печени, independent recapitulation unpublished результатов по антимикробной резистентности за 2 дня.

**Ограничения:** Знания ограничены open-access литературой, нет доступа к негативным результатам, возможны hallucination. Безопасность: проверка на входе и выходе, Trusted Tester Program.

## Практический пример кода: Agent Laboratory

Agent Laboratory — фреймворк для автономного научного исследования (MIT License). Включает AgentRxiv — децентрализованный репозиторий для совместного использования исследований.

Этапы:
1. **Обзор литературы:** LLM-агенты собирают и анализируют научные публикации (arXiv).
2. **Эксперименты:** Формулирование дизайна, подготовка данных, выполнение, анализ (Python, Hugging Face).
3. **Написание отчёта:** Автоматическая генерация научных отчётов (LaTeX).
4. **Обмен знаниями:** AgentRxiv для накопительного прогресса.

**Механизм оценки (Judgment):** Три агента-рецензента с разными перспективами:

```python
class ReviewersAgent:
    def inference(self, plan, report):
        review_1 = get_score(plan, report, reviewer_type="harsh but fair, expects good experiments")
        review_2 = get_score(plan, report, reviewer_type="harsh, critical, looks for impactful ideas")
        review_3 = get_score(plan, report, reviewer_type="harsh, open-minded, looks for novel ideas")
        return f"Reviewer #1:\n{review_1}\nReviewer #2:\n{review_2}\nReviewer #3:\n{review_3}"
```

Функция `get_score` генерирует detailed review в формате JSON с полями: Summary, Strengths, Weaknesses, Originality, Quality, Clarity, Significance, Questions, Limitations, Overall (1-10), Decision (Accept/Reject).

**Агенты в иерархии:**

* **Professor Agent:** Устанавливает исследовательскую повестку, делегирует задачи, генерирует README.
* **PostDoc Agent:** Выполняет исследование: обзор литературы, дизайн экспериментов, интерпретация результатов, написание статей.
* **Reviewer Agents:** Критическая оценка научной строгости (peer review).
* **ML Engineering Agents:** Помощь PhD-студенту в написании кода для подготовки данных.
* **SW Engineer Agents:** Направление ML-инженера в создании простого кода подготовки данных.

## Краткий обзор

**Что:** AI-агенты работают в рамках предопределённых знаний, что ограничивает способность решать новые задачи. Нужен переход от оптимизации к проактивному исследованию.

**Почему:** Многоагентные системы со специализированными LLM эмулируют научный метод: генерация гипотез, критический обзор, эволюция идей. Google Co-Scientist, Agent Laboratory.

**Когда использовать:** В открытых, сложных, быстро эволюционирующих доменах. Для задач, требующих новых гипотез, стратегий, инсайтов.

## Ключевые выводы

* Исследование и открытие позволяют агентам активно находить новую информацию в динамичных средах.
* Google Co-Scientist демонстрирует автономную генерацию гипотез и дизайн экспериментов.
* Agent Laboratory автоматизирует обзор литературы, эксперименты и написание отчётов через специализированные роли.
* Агенты расширяют человеческую креативность, ускоряя инновации.

## Заключение

Паттерн «Исследование и открытия» — суть подлинно агентной системы, определяющей способность переходить от пассивного выполнения инструкций к проактивному исследованию среды. Многоагентные фреймворки (Google Co-Scientist, Agent Laboratory) создают иерархии, имитирующие человеческие исследовательские команды. Оркестрация emergent-поведений позволяет системе преследовать долгосрочные цели с минимальным вмешательством человека. Это трансформирует вычислительные инструменты в независимых партнёров в поиске знания. Развитие таких мощных агентных возможностей требует приверженности к безопасности и этическому надзору.
