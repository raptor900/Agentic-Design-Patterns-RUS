# Глава 11: Постановка целей и мониторинг

Чтобы AI-агенты были по-настоящему эффективными и целенаправленными, им нужно нечто большее, чем способность обрабатывать информацию или использовать инструменты; им нужен чёткий вектор движения и способность понимать, достигают ли они успеха. Именно здесь в действие вступает паттерн «Постановка целей и мониторинг». Он заключается в предоставлении агентам конкретных целевых установок и оснащении их средствами отслеживания прогресса и определения достижения поставленных целей.

## Обзор паттерна «Постановка целей и мониторинг»

Представьте планирование поездки. Вы не просто оказываетесь в пункте назначения. Вы решаете, куда хотите попасть (целевое состояние), определяете, откуда начинаете (начальное состояние), рассматриваете доступные варианты (транспорт, маршруты, бюджет), а затем намечаете последовательность шагов: забронировать билеты, собрать вещи, добраться до аэропорта/вокзала, сесть на транспорт, прибыть, найти жильё и т.д. Этот пошаговый процесс, часто учитывающий зависимости и ограничения, — по сути то, что мы понимаем под планированием в агентных системах.

В контексте AI-агентов планирование обычно включает агент, принимающий высокоуровневую цель и автономно или полуавтономно генерирующий серию промежуточных шагов или подцелей. Эти шаги затем выполняются последовательно или в более сложном потоке, потенциально включающем другие паттерны: использование инструментов, маршрутизацию или многоагентное взаимодействие. Механизм планирования может включать сложные алгоритмы поиска, логическое рассуждение или — всё чаще — использование возможностей больших языковых моделей (LLM) для генерации правдоподобных и эффективных планов на основе обучающих данных.

Хорошая способность к планированию позволяет агентам решать задачи, выходящие за рамки простых одношаговых запросов. Она даёт возможность обрабатывать многоаспектные запросы, адаптироваться к изменяющимся обстоятельствам через перепланирование и оркестрировать сложные рабочие процессы. Это базовый паттерн, лежащий в основе многих продвинутых агентных поведений, превращающий простую реактивную систему в способную проактивно работать к определённой цели.

## Практические применения и сценарии использования

Паттерн «Постановка целей и мониторинг» необходим для построения агентов, способных работать автономно и надёжно в сложных реальных сценариях:

* **Автоматизация клиентской поддержки:** Цель агента — «решить вопрос клиента по биллингу». Агент отслеживает диалог, проверяет записи в БД и использует инструменты для корректировки счетов. Успех контролируется подтверждением изменения и положительной обратной связью клиента. Если проблема не решена — эскалация.
* **Персонализированные обучающие системы:** Цель — «улучшить понимание учеником алгебры». Агент отслеживает прогресс по упражнениям, адаптирует материалы и метрики (точность, время выполнения), корректируя подход при затруднениях.
* **Ассистенты управления проектами:** Задача — «обеспечить завершение вехи X к дате Y». Агент отслеживает статусы задач, коммуникации команды и доступность ресурсов, сигнализируя о задержках и предлагая корректирующие действия.
* **Автоматизированные торговые боты:** Цель — «максимизировать доходность портфеля в пределах допустимого риска». Агент непрерывно отслеживает рыночные данные, стоимость портфеля и индикаторы риска, корректируя стратегию при нарушении порогов.
* **Робототехника и беспилотники:** Основная цель — «безопасно доставить пассажиров из A в B». Агент постоянно отслеживает окружение, собственное состояние и прогресс маршрута, адаптируя поведение.
* **Модерация контента:** Цель — «выявлять и удалять вредоносный контент». Агент отслеживает входящий контент, применяет модели классификации и метрики (ложные срабатывания), корректируя фильтры.

Этот паттерн фундаментален для агентов, которым необходимо работать надёжно, достигать конкретных результатов и адаптироваться к динамическим условиям.

## Практический пример кода

Для иллюстрации паттерна «Постановка целей и мониторинг» рассмотрим пример с использованием LangChain и OpenAI API. Скрипт описывает автономного AI-агента, генерирующего и дорабатывающего Python-код. Его основная функция — создание решений для поставленных задач с соблюдением пользовательских критериев качества.

Агент использует паттерн «постановка целей и мониторинг»: он не просто генерирует код один раз, а входит в итеративный цикл создания, самооценки и улучшения. Успех определяется AI-оценкой: соответствует ли сгенерированный код поставленным целям. Финальный результат — отшлифованный, прокомментированный файл.

```python
import os
import random
import re
from pathlib import Path
from langchain_openai import ChatOpenAI
from dotenv import load_dotenv, find_dotenv

_ = load_dotenv(find_dotenv())
OPENAI_API_KEY = os.getenv("OPENAI_API_KEY")
if not OPENAI_API_KEY:
    raise EnvironmentError("Please set the OPENAI_API_KEY environment variable.")

llm = ChatOpenAI(model="gpt-4o", temperature=0.3, openai_api_key=OPENAI_API_KEY)


def generate_prompt(
    use_case: str, goals: list[str], previous_code: str = "", feedback: str = ""
) -> str:
    base_prompt = f"""
You are an AI coding agent. Your job is to write Python code based on the following use case:
Use Case: {use_case}

Your goals are:
{chr(10).join(f"- {g.strip()}" for g in goals)}
"""
    if previous_code:
        base_prompt += f"\nPreviously generated code:\n{previous_code}"
    if feedback:
        base_prompt += f"\nFeedback on previous version:\n{feedback}\n"
    base_prompt += "\nPlease return only the revised Python code. Do not include comments or explanations outside the code."
    return base_prompt


def get_code_feedback(code: str, goals: list[str]) -> str:
    feedback_prompt = f"""
You are a Python code reviewer. A code snippet is shown below. Based on the following goals:
{chr(10).join(f"- {g.strip()}" for g in goals)}

Please critique this code and identify if the goals are met. Mention if improvements are needed for clarity, simplicity, correctness, edge case handling, or test coverage.

Code:
{code}
"""
    return llm.invoke(feedback_prompt)


def goals_met(feedback_text: str, goals: list[str]) -> bool:
    review_prompt = f"""
You are an AI reviewer.
Here are the goals:
{chr(10).join(f"- {g.strip()}" for g in goals)}

Here is the feedback on the code:
\"\"\"
{feedback_text}
\"\"\"

Based on the feedback above, have the goals been met? Respond with only one word: True or False.
"""
    response = llm.invoke(review_prompt).content.strip().lower()
    return response == "true"


def clean_code_block(code: str) -> str:
    lines = code.strip().splitlines()
    if lines and lines[0].strip().startswith("```"):
        lines = lines[1:]
    if lines and lines[-1].strip() == "```":
        lines = lines[:-1]
    return "\n".join(lines).strip()


def add_comment_header(code: str, use_case: str) -> str:
    comment = f"# This Python program implements the following use case:\n# {use_case.strip()}\n"
    return comment + "\n" + code


def save_code_to_file(code: str, use_case: str) -> str:
    summary_prompt = (
        f"Summarize the following use case into a single lowercase word or phrase, "
        f"no more than 10 characters, suitable for a Python filename:\n\n{use_case}"
    )
    raw_summary = llm.invoke(summary_prompt).content.strip()
    short_name = re.sub(r"[^a-zA-Z0-9_]", "", raw_summary.replace(" ", "_").lower())[:10]
    random_suffix = str(random.randint(1000, 9999))
    filename = f"{short_name}_{random_suffix}.py"
    filepath = Path.cwd() / filename
    with open(filepath, "w") as f:
        f.write(code)
    return str(filepath)


def run_code_agent(use_case: str, goals_input: str, max_iterations: int = 5) -> str:
    goals = [g.strip() for g in goals_input.split(",")]
    previous_code = ""
    feedback = ""

    for i in range(max_iterations):
        prompt = generate_prompt(
            use_case, goals, previous_code,
            feedback if isinstance(feedback, str) else feedback.content,
        )
        code_response = llm.invoke(prompt)
        raw_code = code_response.content.strip()
        code = clean_code_block(raw_code)

        feedback = get_code_feedback(code, goals)
        feedback_text = feedback.content.strip()

        if goals_met(feedback_text, goals):
            break
        previous_code = code

    final_code = add_comment_header(code, use_case)
    return save_code_to_file(final_code, use_case)


if __name__ == "__main__":
    use_case_input = "Write code to find BinaryGap of a given positive integer"
    goals_input = "Code simple to understand, Functionally correct, Handles comprehensive edge cases, Takes positive integer input only, prints the results with few examples"
    run_code_agent(use_case_input, goals_input)
```

Представьте скрипт как автономного AI-программиста, получившего проектное задание (см. Рис. 1). Вы даёте ему подробное техзадание — конкретную задачу и строгий чеклист качества («решение должно быть простым», «функционально корректным», «обрабатывать неожиданные случаи»).

Рис. 1: Пример постановки целей и мониторинга

AI-программист создаёт первый вариант, затем проводит строгую самооценку: сравнивает работу с каждым пунктом чеклиста, вынося вердикт «True» или «False». При «False» агент не сдаётся — входит в фазу доработки, используя инсайты для исправления. Цикл черновик → самооценка → доработка продолжается до «True» или достижения лимита попыток.

**Оговорки и соображения:** Это демонстрационный пример, а не production-код. LLM может неправильно понять цель и ошибочно оценить свой успех как положительный. Модель может hallucinate. Когда одна и та же LLM и пишет код, и оценивает качество, ей сложнее обнаружить неверное направление. В конечном счёте, LLM не производят безупречный код магически — код всё равно нужно запускать и тестировать.

Более надёжный подход — разделение ответственности между несколькими агентами. Например:

* Peer Programmer: помогает писать код.
* Code Reviewer: выявляет ошибки, предлагает улучшения.
* Documenter: генерирует документацию.
* Test Writer: создаёт unit-тесты.
* Prompt Refiner: оптимизирует взаимодействие с AI.

В такой многоагентной системе Code Reviewer как отдельная сущность значительно повышает объективность оценки.

## Краткий обзор

**Что:** AI-агенты часто лишены чёткого направления, что не позволяет им работать целенаправленно за пределами простых реактивных задач. Без определённых целей они не могут самостоятельно решать сложные многошаговые задачи.

**Почему:** Паттерн «Постановка целей и мониторинг» встраивает целенаправленность и самооценку в агентные системы. Он включает определение чётких измеримых целей и механизма непрерывного отслеживания прогресса. Это создаёт петлю обратной связи, позволяющую агенту оценивать производительность и корректировать курс.

**Когда использовать:** Когда AI-агент должен автономно выполнять многошаговые задачи, адаптироваться к динамическим условиям и надёжно достигать высокоуровневые цели без постоянного вмешательства человека.

## Ключевые выводы

* Паттерн «Постановка целей и мониторинг» оснащает агентов целенаправленностью и механизмами отслеживания прогресса.
* Цели должны быть конкретными, измеримыми, достижимыми, релевантными и ограниченными по времени (SMART).
* Чёткое определение метрик и критериев успеха необходимо для эффективного мониторинга.
* Мониторинг включает наблюдение за действиями агента, состоянием среды и выводом инструментов.
* Петли обратной связи от мониторинга позволяют агентам адаптироваться, пересматривать планы или эскалировать проблемы.
* В Google ADK цели задаются через инструкции агента, мониторинг — через управление состоянием и взаимодействие с инструментами.
