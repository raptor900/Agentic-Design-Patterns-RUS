# Приложение G — Кодирующие агенты

## Vibe Coding: отправная точка

«Vibe coding» стал мощной техникой для быстрой инновации и творческого исследования. Практика включает использование LLM для генерации начальных черновиков, наброска сложной логики или быстрых прототипов. Vibe coding особенно эффективен при исследовании незнакомых API или тестировании архитектурных паттернов. Сгенерированный код — творческий катализатор, основа для критики, рефакторинга и расширения.

## Агенты как члены команды

Индустрия смещается от raw code generation к интегрированной парадигме. Эффективные команды не просто делегируют задачи агентам — они расширяют себя набором специализированных coding agents. Агенты — неутомимые специализированные члены команды.

Sundar Pichai (2025): «Более 30% нового кода в Google assisted или сгенерировано Gemini». Microsoft — аналогично.

Три принципа:

1. **Человек-оркестратор:** Разработчик — team lead и архитектор. Управляет workflow, ставит цели, принимает финальные решения. Агенты — мощные, но вспомогательные.
2. **Первичность контекста:** Производительность агента зависит от качества контекста. Разработчик собирает «брифинг»: полный код, внешняя документация, чёткий human brief.
3. **Прямой доступ к моделям:** Frontier-модели (Gemini 2.5 Pro, Claude Opus 4) без intermediary платформ.

## Основные компоненты

**Оркестратор — человек-разработчик:** Определяет задачи, готовит контекст, валидирует работу агентов.

**Область подготовки контекста:** Рабочее пространство для каждой задачи (task-context/) с файлами целей, кода и документации.

**Специализированные агенты:**

* **Scaffolder (Исполнитель):** Пишет код, реализует фичи, создаёт boilerplate. Промпт: «Ты — senior software engineer. Реализуй фичу на основе 01_BRIEF.md и паттернов 02_CODE/.»
* **Test Engineer (Контроль качества):** Пишет unit/integration/e2e тесты. Промпт: «Напиши полный набор тестов на pytest для 02_CODE/.»
* **Documenter (Скриб):** Генерирует документацию. Промпт: «Сгенерируй markdown-документацию для API-эндпоинтов.»
* **Optimizer (Рефакторинг-партнёр):** Предлагает оптимизации и рефакторинг. Промпт: «Проанализируй код на bottlenecks, предложи конкретные улучшения.»
* **Process Agent (Супервизор кода):** Critique → Reflection. Первая проходка выявляет проблемы, затем синтезирует и приоритизирует находки.

## Практическая реализация

### Чеклист настройки

1. **Доступ к frontier-моделям:** API-ключи для Gemini 2.5 Pro и Claude 4 Opus (dual-provider).
2. **Локальный context orchestrator:** CLI-инструмент с context.toml для прозрачного контроля контекста.
3. **Version-controlled prompt library:** /prompts директория в Git с invocation-промптами (reviewer.md, documenter.md, tester.md).
4. **Git hooks для workflow:** pre-commit hook для автоматического вызова Reviewer Agent.

### Принципы руководства командой

* **Сохраняйте архитектурное владение:** Вы определяете «что» и «зачем», агенты — «как». Финальный арбитр дизайна.
* **Мастерство брифа:** Качество вывода = качество ввода. Полный, чёткий, однозначный контекст.
* **Финальный quality gate:** Вывод агента — всегда предложение. Вы — финальный фильтр.
* **Итеративный диалог:** Лучшие результаты — из разговора, не монолога.

## Заключение

Будущее разработки — аугментированное. Эра одинокого кодера уступила парадигме, где разработчики руководят командами специализированных AI-агентов. Модель не умаляет человеческую роль — возвышает её. Перемещая рутинные задачи на агентов, разработчики концентрируются на стратегических инновациях и архитектуре. Фундаментальное отношение переопределено: не человек vs. машина, а партнёрство человеческой изобретательности и ИИ.
