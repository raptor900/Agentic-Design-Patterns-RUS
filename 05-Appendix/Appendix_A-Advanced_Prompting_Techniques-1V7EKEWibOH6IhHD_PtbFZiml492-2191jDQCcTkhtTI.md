# Приложение A: Продвинутые техники промптинга

## Введение в промптинг

Промптинг — основной интерфейс взаимодействия с языковыми моделями. Процесс создания входных данных для направления модели к генерации желаемого результата. Включает структурирование запросов, предоставление контекста, указание формата вывода и демонстрацию ожидаемых типов ответов. Хорошо спроектированные промпты максимизируют потенциал моделей. Плохо спроектированные — приводят к неоднозначным или ошибочным результатам.

Цель prompt engineering — consistently получать высококачественные ответы. Требуется понимание возможностей и ограничений моделей и эффективная коммуникация целей.

## Базовые принципы промптинга

**Ясность и конкретность:** Инструкции должны быть однозначными и точными. Модели интерпретируют паттерны; множественные интерпретации ведут к непреднамеренным ответам. Определяйте задачу, формат вывода, ограничения.

**Краткость:** Конкретность не должна жертвовать краткостью. Инструкции должны быть прямыми. Сложные конструкции запутывают модель.

**Использование глаголов:** Выбор глагола — ключевой инструмент. Эффективные: Analyze, Categorize, Classify, Compare, Create, Describe, Evaluate, Extract, Generate, Identify, List, Organize, Summarize, Translate, Write.

**Инструкции вместо ограничений:** Позитивные инструкции эффективнее негативных ограничений.

**Экспериментация и итерация:** Prompt engineering — итеративный процесс. Черновик, тест, анализ, доработка.

## Базовые техники промптинга

### Zero-Shot (без примеров)

Самая простая форма: инструкция + данные без примеров. Зависит от предобучения модели.

* **Когда:** Задачи, знакомые модели (вопросы, обобщение).
* **Пример:** Переведите на французский: "Hello, how are you?"

### One-Shot (один пример)

Один пример входа и желаемого выхода перед задачей.

* **Когда:** Нестандартный формат вывода.
* **Пример:** English: "Thank you." -> Spanish: "Gracias." / English: "Please." -> Spanish:

### Few-Shot (несколько примеров)

Три-пять примеров. Улучшает воспроизводимость.

* **Когда:** Задачи со специфическим форматом, классификацией.
* **Важность качества:** Точность, репрезентативность, покрытие edge-cases.
* **Перемешивание классов:** Для классификации — перемешивать порядок примеров.
* **Many-Shot:** Современные LLM эффективно используют сотни примеров.

## Структурирование промптов

### System Prompting

Задаёт общий контекст и поведение модели. Определяет тон, стиль, правила. Также для safety.

* **Пример:** "Вы — полезный и безвредный AI-ассистент. Отвечайте вежливо."

### Role Prompting

Назначение персонажа. Задаёт тон, стиль, экспертизу.

* **Пример:** "Вы — опытный travel blogger. Напишите абзац о скрытых местах Рима."

### Разделители

Разделители визуально разделяют инструкции, контекст, примеры, вход. Минимизирует misinterpretation.

## Инжиниринг контекста

Динамическое предоставление фоновой информации. Фундаментален для памяти, решений, координации агентов. Качество контекста важнее архитектуры модели.

Слои контекста:
* System prompts: базовые инструкции
* Внешние данные: извлечённые документы, выводы инструментов
* Неявные данные: идентичность пользователя, история, состояние среды

## Структурированный вывод

JSON, XML, CSV для machine-readable формата. JSON force model создавать структуру, limit hallucination.

Pydantic для валидации: model_validate_json — парсинг + валидация за один шаг.

# Техники рассуждения

## Chain of Thought (CoT)

Явное направление модели генерировать промежуточные шаги. "Подумай пошагово."

* **Zero-Shot CoT:** "Let's think step by step" без примеров.
* **Few-Shot CoT:** Примеры с пошаговыми рассуждениями.

Преимущества: low-effort, интерпретируемость, робастность. Недостаток: удлинение вывода, рост стоимости.

## Self-Consistency

Несколько путей рассуждения, majority vote. Улучшает точность, но умножает стоимость.

## Step-Back Prompting

Сначала общий принцип, затем конкретная задача с контекстом.

## Tree of Thoughts (ToT)

Несколько путей рассуждения одновременно, древовидная структура. Для сложных задач с backtrack.

# Техники действия и взаимодействия

## Использование инструментов

Агент использует внешние инструменты. Модель генерирует JSON-вызов. Система выполняет и возвращает результат.

## ReAct (Рассуждение + Действие)

Цикл: Мысль, Действие, Наблюдение, Мысль... до завершения задачи. Позволяет агенту dynamically собирать информацию и реагировать на выводы инструментов.

# Продвинутые техники

## Автоматический prompt engineering (APE)

LLM генерирует, оценивает и дорабатывает промпты.

## Итеративный промптинг

Человеко-ориентированный цикл доработки.

## Отрицательные примеры

Показ нежелательного вывода для clarification.

## Аналогии

Объяснение задачи через знакомое сравнение.

## Декомпозиция

Разбиение сложной задачи на подзадачи.

## RAG

Доступ к внешней актуальной информации. Поиск в БД, контекст в промпт, grounded ответ.

## Persona Pattern

Описание целевой аудитории. "Объясните квантовую физику для школьника."

## Google Gems

Пользовательские экземпляры Gemini со специализированными инструкциями.

## Meta-подход

LLM анализирует и улучшает промпты. Accelerates итерацию.

## Код и мультимодальность

Промптинг для генерации, объяснения, отладки кода. Мультимодальный: изображение + текст.

## Best practices

Примеры, простота, конкретность, формат вывода, документирование, автоматические тесты.

## Заключение

Prompts — инженерная дисциплина. Базовые принципы — фундамент. Структурирование — контроль. CoT, ToT — декомпозиция. JSON + Pydantic — predictability. ReAct — действие. RAG и контекст — grounding. Мастерство промптинга превращает генератор текста в sophisticated автономного агента.
